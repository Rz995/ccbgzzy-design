# 配色系统 · 同色阶 + 单点强调

> v1.1 核心思想：**统一靠同色阶，重点靠单点强调。**
> 一个页面只围绕"一种主色相"展开，层级用**明度台阶**(seq-1…5)拉开；
> 整页只保留**一个**暖色强调 `--accent`，落在最该被看到的那一处。
> 这就是参考图里那种科研期刊式的高级感——克制、统一、又有焦点。
> 颜色代表角色，不代表色值：换主题只换一组值，角色关系永远不变。

## 一、语义角色表

| Token | 角色 | 典型用途 |
|-------|------|----------|
| `--bg` / `--surface` / `--surface-2` | 背景三层 | 底 / 卡片 / 悬浮·次级面板 |
| `--title` | 标题（最高权重） | h1–h4、KPI 数值、表格首列 |
| `--fg` | 正文 | 段落、表格单元 |
| `--fg-2` | 次要文字 | 描述、副标题 |
| `--fg-muted` | 辅助文字 | 标签、页脚、占位 |
| `--primary` / `--primary-strong` | 主色 / 亮版 | 链接、主按钮、激活态、重点词、进度条 |
| `--accent` / `--accent-soft` | **唯一暖色强调** | 整页 1–2 处焦点（最关键数字、唯一主行动） |
| `--seq-1 … --seq-5` | **同色阶台阶**（浅→深） | 多类别 / 排序 / 分组 / 多系列图表 |
| `--border` / `--border-strong` | 边框 / 实边框 | 卡片线 / 表头下线 |
| `--success` `--warning` `--danger` `--info` | 状态（语义固定） | 上涨成功 / 注意 / 下跌错误 / 提示 |

## 二、铁律（违反即"一眼 AI"）

1. **禁止硬编码。** 内容里只能 `var(--token)`；审查全局搜 `#` 和 `rgba(`，除 token 定义外不应出现。
2. **多类别不用多色相，用同色阶。** 三类证书、多组对比、分级标签——一律用 `--seq-1..5`
   的明度台阶（浅→深），而不是蓝/绿/橙各一色。这是这次返工的关键。
3. **`--accent` 是稀缺资源。** 整页最多一两处。满屏强调=没有重点。多了就降级用 `--seq-*` 或 `--primary`。
4. **三层文字 + 主色。** 文字不超过 `--title/--fg/--fg-2(--fg-muted)`；重点 `--primary`，唯一焦点 `--accent`。
5. **状态色不做装饰。** `--success/--warning/--danger/--info` 只表达状态；类别区分交给 `--seq-*`。
6. **同色阶取值要拉开。** 多类别从 `--seq-*` 选时优先选间隔大的（如 seq-1/seq-3/seq-5），相邻台阶对比太弱。
7. **对比度达标。** 正文/标题对背景 ≥ 4.5:1，大字 ≥ 3:1；当前 7 个 theme key 已校准。
8. **禁止任何渐变。** 全站 0 渐变——背景、文字、按钮、卡片、进度条、图表一律纯色。
   `linear-gradient`/`radial-gradient`/`conic-gradient`/背景扫光一概不用。要纵深用"纯色台阶"
   （bg→surface→surface-2）和边框，不要用渐变。
9. **seq 只用于"填充"，不当文字色。** `--seq-*` 给进度条、柱/块、图表区域、KPI 顶部细条上色；
   它的深端在深色主题里当文字会糊（看不清）。**文字永远用 `--title/--fg/--fg-2/--primary/--accent`**，
   类别区分放在"填充色/细条"上，不要去染数字或正文。

## 三、"统一中有重点"怎么落地（对照参考图）

- 整页大面积是同一色相的不同明度（背景、卡片、表格、图表台阶）——形成**统一基调**。
- 只在一个地方用 `--accent` 暖色——比如"最关键的那个 KPI 数字"或"唯一的主按钮"——形成**焦点**。
- 例：多个并列指标里，几个同级类别用 seq-2/seq-3/seq-4（同色相三个深浅）区分，而"最重要的那个总指标"用 `--accent` 点亮。一眼就知道看哪。

## 四、主题（同色阶 + 单点强调）

**默认 `sage`（砂绿）** — 奶油底 + 鼠尾草同色阶 + 标志橙强调 + #1A1A1A 文字（仅 light，无深色配对）。
当前 **7 个 theme key** 全部开放可切换：
`sage`（默认）、`gov-finance-blue`/`gov-finance-blue-dark`、`sky-field`/`sky-field-dark`、`summer-coast`、`warm-sand`。
配对深浅（`CCBGZZY_toggleMode()`）的是 `gov-finance-blue` 与 `sky-field`；其余仅 light。
各主题完整 token、适合/不适合场景、注意事项见 [experimental-themes.md](experimental-themes.md)。
新增/修改主题见下节。

## 五、新增/修改主题

只改 `theme-config.js`：① `CCBGZZY_THEMES` 加 key，填满全部 token（含 `--seq-1..5`、`--accent`）并标 `mode`；
② `CCBGZZY_THEME_META` 加显示信息，`CCBGZZY_THEME_ORDER` 排序；③ 如需深浅配对，在 `CCBGZZY_toggleMode` 的 `pair` 登记。
模板/页面读取 `CCBGZZY_getThemeOptions()`，不要手写主题列表。
