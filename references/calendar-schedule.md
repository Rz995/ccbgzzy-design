# calendar / schedule 页面专项规则

> 适用：月度日历、营销日历、活动排期、任务排期、内容排期、排班。
> 核心规则：**月历是导航和摘要，不是明细容器。换行是信息过载信号，不是默认排版方式。**

月历格子不是内容容器，而是时间索引。凡是需要解释、换行、多个标签、多字段展示的内容，都必须离开月历格子，进入时间轴、周节奏、列表或详情层。

## 一、先分页面变体

### `calendar-report` 汇报型日历

适用：给领导、组长、运营负责人看。

目标：看懂本月营销节奏、关键节点、资源安排和风险提醒。

结构：

1. 第一屏：一句话结论 + 3-5 个关键节点 + 本月节奏。
2. 第二屏：关键节点时间轴 / 周节奏卡。
3. 第三屏：轻量月历。
4. 详情：每日任务折叠、抽屉或附录。

汇报版看节奏，不查明细。主题切换器、调试控件、无关工具层默认隐藏。

### `calendar-execution` 执行型日历

适用：给运营同事、客户经理照着执行。

目标：查每天做什么、发什么、跟进什么。

结构：

1. 顶部：本周 / 本月任务摘要。
2. 主体：周视图或日期列表。
3. 每日详情：客群、产品、渠道、话术、负责人、状态。
4. 支持筛选、折叠、导出。

执行型可以展示更多明细，但仍然不能让 7×5 月历格子承载完整执行清单。

## 二、汇报型月历硬规则

`calendar-report` 的月历只展示关键节奏，不展示完整执行清单。

- 普通日期最多 0-1 条短事项。
- 关键日期最多 1 个标签 + 1 行短文案。
- 每个日期格最多 12 个中文字。
- 日期格内禁止 `flex-wrap`。
- 日期格内禁止自然多行换行。
- 多任务统一显示 `+N 项` 或 `更多`，不展开全文。
- 重复任务上提到顶部节奏区或周节奏区。
- 周末 / 非工作日弱化，不占同等视觉权重。
- 主题切换器、调试控件、无关工具层在汇报版默认隐藏。

推荐 CSS 类：外层加 `.cal-report`，并使用 `.cal-task-title` 包裹任务标题。

```html
<div class="cal-report">
  <div class="cal-day is-key">
    <div class="cal-date">15</div>
    <div class="cal-task">
      <span class="cal-tag tag-cat">发薪</span>
      <span class="cal-task-title">代发节点</span>
    </div>
    <div class="cal-more">+3 项</div>
  </div>
</div>
```

## 三、月历主视图禁用条件

满足任一条件时，不得使用 7×5 月历作为主体：

- 平均每天超过 1 条任务。
- 超过 20% 的日期有 2 条以上任务。
- 单日任务需要展示客群、产品、渠道、话术、负责人等多字段。
- 多数日期文案超过 12 个中文字。
- 同一日存在多个业务线并行。

触发后自动改结构：

- 汇报版：关键节点时间轴 + 周节奏卡 + 轻量月历导航。
- 执行版：周视图 / 日期列表 / 可筛选任务表。
- 查询版：筛选 + 表格 + 折叠详情。

## 四、换行契约

### 禁止换行

以下区域必须默认 `nowrap + ellipsis`，最多 1 行：

- 日期格内任务标题。
- 标签 / 徽章 / 状态角标。
- 按钮 / 导航 / 表头。
- KPI 数字和单位。
- 卡片标题第一行。

### 允许换行

以下区域可以换行，但必须控制行数：

- 摘要段落。
- 详情说明。
- 每日执行详情。
- 折叠面板正文。
- 表格移动端卡片正文。

普通说明最多 2-3 行；长说明进入详情层。

### 换结构触发

出现下列任一情况，月历不能作为主展示：

- 一个日期格需要超过 1 行任务文案。
- 一个格子里出现 2 个以上标签。
- 连续多天都需要换行。
- 同一日任务超过 1 条且都需要解释。

## 五、日历颜色预算

`calendar-report` 单页可见颜色角色最多 3 类：

1. 主色：普通关键节点 / 当前主线。
2. 强调色：全月最重要节点，最多 3-5 个日期。
3. 灰色：普通日期 / 非工作日 / 次要信息。

限制：

- 状态色 `danger` / `warning` / `success` / `info` 默认不进入月历格子，除非页面主题就是风险监控。
- 同一标签类型必须同一种颜色。
- 不允许同一层级里有的标签彩色、有的标签无色。
- 如果颜色无法表达清晰语义，宁可不用颜色，用位置、粗细、分组表达。
- 标签不是装饰，只能表达业务类别、状态、节点类型三者之一；同一层级最多使用一种标签体系。

## 六、重复目标与 AB-test

- 每日相同目标不要在每个格子里重复全文；放到顶部节奏区或周节奏区，格子里只留短标记。
- AB-test 顶部只保留一句摘要；详细规则放附录、折叠区或抽屉。
- 客群、产品、话术、负责人、渠道等多字段永远不进汇报型月历格子。

## 七、组件与 CSS 要求

基础组件见 `assets/base.css`：

- `.cal-grid` 月历网格。
- `.cal-day` 日期格，`.is-key` 关键日，`.is-dim` 非本月，`.is-weekend` 非工作日。
- `.cal-date` 日期。
- `.cal-task` 短事项。
- `.cal-task-title` 单行标题。
- `.cal-more` 显示 `+N 项` 或 `更多`。
- `.cal-tag` 标签。
- `.keydate-strip` / `.keydate` 关键日期横条。
- `.week-row` 周节奏。
- `.cal-report` 汇报型日历专用容量约束。

`calendar-report` 专用样式必须满足：

- `.cal-report .cal-task` 禁止 `flex-wrap: wrap`。
- `.cal-report .cal-task-title` 使用 `white-space: nowrap; overflow: hidden; text-overflow: ellipsis;`。
- `.cal-report .cal-day` 不允许因为内容变多无限增高。
- `.cal-report .cal-tag` 只允许一个，且 `white-space: nowrap`。
- `.cal-report .cal-more` 使用 `+N 项` 或 `更多`。
- 普通日期弱边框或无边框，关键日期才使用强调边框。
- 非工作日使用浅灰背景或降低透明度，不与工作日同等抢眼。

## 八、验收句

交付前逐条自问：

- 月历格子是不是只做时间索引？
- 是否存在需要解释、换行、多个标签、多字段展示的日期？
- 如果有，这些内容是否已进入时间轴、周节奏、列表或详情层？
- 汇报版是否能一眼看节奏，执行版是否能查明细？

答案有一项不清楚，就不要继续把内容塞进月历格子。
