# calendar / schedule 页面专项规则

> 适用：月度日历、营销日历、活动排期、任务排期、内容排期、排班。
> 这类页面**不套普通 dashboard**，必须三层展示。核心病根是"把每天的全部细节塞进日期格子"——本篇专治。

## 一、三层展示（必须）

| 层 | 放什么 | 不放什么 |
|----|--------|----------|
| ① 月度总览 | 本月一句话重点、月目标、工作日数、关键节点数、执行节奏 | 不放逐日明细 |
| ② 月历视图 | 每个日期**只放 1–3 条核心任务**（标题 + 最多 2 个标签） | 不放完整客群/产品/AB-test/文案 |
| ③ 详情视图 | 完整客群、产品、AB-test 规则、朋友圈文案 → **折叠区 / 侧栏 / 附录** | 不进月历格子、不占第一屏 |

## 二、日期格子克制
- 每天任务 **> 3 条**：默认只显示前 3 条 + `更多 N 项`（点击展开详情层）。
- 格子里每条任务 = 一行标题 + 至多 2 个标签；不写长描述。

## 三、重复目标上提
- 每日相同的目标（如"日均 X")**不要在每个格子里重复全文**；放到顶部或**周头部**，格子里只留简短标记（如一个小圆点/缩写）。

## 四、AB-test 不占首屏
- 顶部只保留**一句摘要**（如"本月 A/B：X 方案对比 Y"）。
- 详细规则放**附录或折叠区**，不作为大块内容长期占据第一屏。

## 五、标签：每个任务最多 2 个
- **主标签（业务类，1 个）**：代发 / CTS / 存款 / 理财 / 月月赢 / 朋友圈 / 联动（按实际业务取）。
- **状态标签（1 个）**：AB / 重叠 / 到期 / 端午 / 重点（按实际取）。
- **禁止标签堆叠**（一个任务挂 3 个以上标签）。
- 颜色：主标签用 `--seq-*` 同色阶区分类别；状态标签用语义色——`重点`→`--accent`(克制，全页 1–2 处)、
  `到期/风险`→`--warning`、严重冲突→`--danger`(仅严重)、节日(端午)→`--info`/`--attention`。**不要每个标签都上高饱和色。**

## 六、关键日期突出
用"关键日期横条"列出，并在月历对应格子加高亮边/角标。典型：
- 发薪日（如 10 / 15 / 20 / 25 号）
- 节假日前最后一个工作日（如端午前）
- 月末冲刺日
> 高亮只用 `--accent` 边或 `--primary` 角标，**不要整格染高饱和色**。

## 七、页面骨架（推荐顺序）
1. 顶部：本月一句话重点（如"6 月：发薪周拉存款 + 端午节点营销"）
2. 关键指标：月目标 / 工作日数 / 关键节点数
3. 本月关键日期横条
4. 周节奏说明（每周主题/打法，一行一周）
5. 月历网格 **compact view**（每格 1–3 条任务 + "更多 N 项"）
6. 本周 / 每周详情（可展开的完整任务）
7. 附录：AB-test 规则、目标口径、产品说明（折叠）

## 八、两个版本
- **管理总览版**：给管理者/组长——重月度总览、关键节点、周节奏；月历更 compact，详情默认折叠。
- **执行详情版**：给运营/执行——支持展开每天完整任务、客群、产品、文案；保留同一视觉与三层结构，只是详情默认更展开。
> 两版共用同一套 token / 组件 / 主题，只在"默认展开程度"和"信息粒度"上分。

## 九、组件（base.css 已提供）
- `.cal-grid` 月历网格、`.cal-day`（`.is-key` 关键日高亮、`.is-dim` 非本月）、`.cal-date`、
  `.cal-task`、`.cal-more`、`.cal-tag`（`.tag-cat` 主标签 / `.tag-state` 状态标签）、
  `.keydate-strip` / `.keydate`、`.week-row` 周节奏。
- 仍遵守：无渐变、语义色边界、字号下限、三层水印、深浅切换。

## 十、通用约束（不变）
砂绿主题默认、无渐变、三层水印、深浅切换、`--accent` 全页 1–2 处、`--danger` 仅严重异常、文字只用中性 title/fg/fg-2/fg-muted。
